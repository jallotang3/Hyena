# Hyena 多面板 VPN 客户端 · 需求分析（V1）

> **文档版本**: v1.2 | **状态**: 评审中 | **更新时间**: 2026-03

---

## 1. 项目背景与产品定位

### 1.1 背景

当前市场上存在多种后端流量管理面板（xboard、v2board、sspanel 等），每个面板生态均依赖配套的客户端完成用户接入。各面板客户端相互独立，难以复用，且大多与单一 UI 风格、单一内核深度耦合，导致：

- 同类产品重复建设，交付成本高。
- 内核升级或面板迁移需大规模重构。
- 品牌化定制（皮肤/LOGO/文案）与业务逻辑混杂，维护困难。

### 1.2 产品定位

**Hyena** 是一款面向 SaaS 运营场景的**多面板可插拔 VPN 客户端**，核心价值主张：

| 维度 | 定位 |
|------|------|
| 面板兼容 | 适配 xboard / v2board / sspanel 等主流面板，通过适配器模式一次集成 |
| 内核扩展 | V1 默认 sing-box，抽象内核驱动接口，未来零侵入替换或并行运行多内核 |
| 品牌定制 | UI 皮肤独立打包，运营商可快速交付品牌化客户端，不触碰业务代码 |
| 自动交付 | GitHub Actions + 配置模板一键生成品牌化多平台安装包 |

### 1.3 V1 核心目标

1. 完成 **xboard** 面板完整商业闭环：注册/登录 → 选购套餐 → 支付 → 订阅连接 → 工单客服。
2. 建立**面板适配器、内核驱动、皮肤系统**三大扩展框架骨架。
3. 输出可执行的 **CI/CD 构建工作流**（至少 Android + Windows）。

## 2. 角色与用户画像

### 2.1 主要角色

| 角色 | 描述 | 核心诉求 | 痛点 |
|------|------|----------|------|
| **终端用户（付费）** | 购买流量套餐的普通付费用户 | 稳定快速连接；套餐/账单透明；遇到问题能快速反馈 | 连接失败不知如何排障；充值/退款流程不清晰 |
| **终端用户（免费/试用）** | 尚未购买或处于试用期的用户 | 快速了解产品能力；低门槛上手；套餐购买路径清晰 | 不知道能买哪些套餐、价格是否合理 |
| **运营/交付人员** | 负责品牌化打包与分发 | 配置简单、构建可重复、制品规范 | 参数散落多处，出包流程不透明 |
| **皮肤开发者** | 为不同品牌定制 UI 外观 | 清晰皮肤合约、快速本地预览 | 不知道改哪里会影响业务 |
| **面板适配开发者** | 接入新面板协议 | 标准接口规范、完善测试用例 | 面板文档缺失，字段语义不一 |
| **内核维护人员** | 负责内核版本升级与适配 | 驱动抽象稳定、版本隔离、可回滚 | 内核 API 变更影响范围不可控 |

### 2.2 核心用户场景

#### UC-01 首次配置与连接（终端用户）

> **前提**：每个站点独立打包，站点地址（API Base URL）与面板类型在构建期通过 CI/CD 模板注入，用户无需手动填写。

1. 用户安装客户端，进入品牌化登录/注册页。
2. 账号密码登录，或通过「注册」创建新账号（含邮件验证码）；Token 自动缓存。
3. 客户端拉取订阅节点，展示节点列表（含延迟）。
4. 用户选择节点（或自动选最低延迟），一键连接，查看状态与流量。

**异常路径**：登录失败 → 弹出错误原因（账号密码错误 / 面板不可达）；节点连接失败 → 提示重试或切换节点。

#### UC-02 日常使用（终端用户）

1. 用户打开客户端（上次状态自动恢复或触发自动连接）。
2. 查看实时流量速率、套餐用量与到期时间。
3. 在节点列表中按延迟或收藏快速切换节点。

#### UC-03 异常容错（终端用户）

| 异常场景 | 客户端行为 |
|---------|-----------|
| Token 过期 | 静默刷新；失败则跳转重新登录 |
| 节点连接失败 | 自动重试（可配置次数）；超限后进入 error 状态，提示手动处理 |
| 面板 API 超时 | 使用本地缓存节点，展示降级提示 |
| 订阅格式异常 | 解析失败的节点跳过，正常节点继续展示，异常计数上报日志 |

#### UC-04 套餐购买与充值（付费用户）

1. 用户在首页看到"流量不足"或"套餐已到期"提示，点击「去购买」。
2. 进入商店页，展示所有可购套餐及计费周期（月付 / 季付 / 年付）。
3. 用户选择套餐与周期，可选填优惠码（自动验证并显示折扣后价格）。
4. 确认订单 → 选择支付方式 → 跳转支付（WebView 或系统浏览器）。
5. 支付完成后轮询订单状态 → 状态更新为"已支付"→ 弹出成功提示，订阅自动更新。

**异常路径**：支付超时 → 保留待支付订单，可在订单中心重新发起结算；优惠码无效 → 提示原因（已过期 / 不适用此套餐 / 已达上限）。

#### UC-05 工单客服（遇到问题的用户）

1. 用户在工单中心点击「新建工单」，填写主题、优先级（普通/高/紧急）、问题描述。
2. 提交后进入工单详情，查看客服回复（实时刷新或下拉刷新）。
3. 用户可继续回复或关闭工单。
4. 客服回复后，用户收到推送通知提醒（如平台支持）。

#### UC-06 邀请好友与佣金（推广用户）

1. 用户进入「推广邀请」页，生成/查看邀请码或邀请链接。
2. 查看邀请统计：已注册人数、累计佣金、待确认佣金、可用佣金余额。
3. 查看佣金明细记录（每笔来源）。
4. 申请将佣金余额转入账户余额，或通过工单提现至外部账户。

#### UC-07 品牌化交付（运营视角）

1. 运营人员准备配置模板（品牌名、包名、面板参数、皮肤 ID、资源包）。
2. 触发 GitHub Actions Workflow（手动 dispatch 或 Git Tag）。
3. 流水线按参数注入，执行平台构建矩阵（Android / Windows / macOS）。
4. 制品按版本命名归档，供分发。

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标值 | 测量条件 |
|------|--------|---------|
| 冷启动到首屏可交互 | ≤ 3s | 非首次安装，主流中端设备 |
| 点击连接到状态变更为 connected | ≤ 5s | 正常网络条件 |
| 单节点延迟测速响应 | ≤ 3s | 超时视为不可用节点 |
| 订阅拉取刷新完成 | ≤ 5s | 正常网络，节点数 ≤ 500 |
| 内存常驻（移动端，含内核） | ≤ 150 MB | Android / iOS |
| 内存常驻（桌面端，含托盘） | ≤ 300 MB | Windows / macOS |

### 4.2 稳定性指标

- **崩溃率**：Crash-free Rate ≥ 99%（接入监控平台后定标）
- **断线自恢复**：网络恢复后自动重连成功率 ≥ 95%（3 次重试内）
- **离线可用**：面板 API 不可达时，使用缓存节点正常连接，展示降级提示
- **面板降级**：面板能力声明不支持某功能时，UI 优雅隐藏，不崩溃

### 4.3 安全要求

- Token / 密钥采用平台安全存储：Android Keystore / iOS Keychain / Windows DPAPI / macOS Keychain。
- TLS 默认严格校验；支持受控证书固定（可配置，适用于自签证书场景）。
- 崩溃日志与导出日志自动脱敏（Token、用户名、明文 IP 不写入日志文件）。
- Android APK 开启 R8 代码混淆；iOS/macOS 开启 Strip Symbols。
- 皮肤包加载前校验合约版本，防止恶意皮肤替换核心组件。

### 4.4 可扩展性

- 新增面板适配器：仅新增 Adapter 实现 + 注册，不改动业务层与 UI 层。
- 替换内核驱动：仅换 EngineDriver 实现 + ConfigBuilder 实现，不改动面板层和皮肤层。
- 新增皮肤包：仅新增皮肤目录并声明 skinId，不改动业务逻辑代码。

### 4.5 可维护性

- 核心模块（适配器、引擎驱动、皮肤管理）代码单元测试覆盖率 ≥ 70%。
- 面板适配器、内核驱动、皮肤包各自独立版本化，可单独升级。
- 关键常量与配置集中管理，禁止硬编码散落业务代码。

### 4.6 国际化与本地化

- V1 支持语言：简体中文（`zh_CN`）、英语（`en`）；语言缺失时自动回退到英语。
- 所有用户可见文案（含错误提示、空状态、Toast）均需纳入翻译，**禁止硬编码中文或英文字符串于业务代码中**。
- 日期/时间展示按 Locale 格式化（如 `2026-03-20` vs `Mar 20, 2026`）；金额展示依面板货币符号而非 Locale 强制。
- RTL（从右到左）布局：V1 不要求，V2 候选。

### 4.7 兼容性要求

| 平台 | 最低版本 | 备注 |
|------|---------|------|
| Android | API 21（Android 5.0） | VPN 权限需 API 21+ |
| iOS | iOS 13.0 | Network Extension 需 iOS 12+ |
| Windows | Windows 10 1903+ | WinTUN 驱动依赖 |
| macOS | macOS 11（Big Sur） | System Extension 需 10.15+ |

## 9. 术语表

| 术语 | 解释 |
|------|------|
| **面板（Panel）** | 后端流量管理系统，如 xboard / v2board / sspanel |
| **面板适配器（PanelAdapter）** | 对接特定面板 API 的抽象实现层，屏蔽面板接口差异 |
| **内核（Engine）** | 负责网络代理与加速的底层程序，V1 为 sing-box |
| **内核驱动（EngineDriver）** | 包装具体内核 FFI/API 的驱动抽象层 |
| **皮肤包（Skin Package）** | 包含主题令牌、资源文件、组件实现的可替换 UI 包 |
| **皮肤合约（Skin Contract）** | 皮肤可定制范围的正式规范，包含版本号、页面清单、槽位清单 |
| **设计令牌（Design Token）** | 颜色、字体、间距、圆角等可配置的最小 UI 变量单元 |
| **订阅（Subscription）** | 包含代理节点信息的 URL 或 API 数据源 |
| **ProxyNode** | 统一节点数据模型，屏蔽各面板与各协议的字段差异 |
| **ViewModel** | 业务层向 UI 层输出的标准化状态与动作，不含样式信息 |
| **制品（Artifact）** | CI/CD 构建输出的安装包文件（APK / EXE / DMG 等） |
